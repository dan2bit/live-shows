WORKFLOW 1: YouTube Studio Channel Playlists → TSV
================================================================================

URL:
  https://studio.youtube.com/channel/UCLk1BKTtXoyFD068_Ob2pcw/content/playlists

PURPOSE:
  Extract all channel playlists (title, description, shareable link, video count)
  into a TSV file. The total count will vary over time; do not hard-code 225.

PRE-REQUISITE — EXPAND ROWS PER PAGE:
  Before starting extraction, set the rows-per-page selector to 50 (the maximum).
  The selector element is:
    <ytcp-select id="page-size">
  If it is not already showing 50, change it. This minimises the number of
  pagination clicks required.

STEP 1 — VERIFY PAGE RANGE:
  After the page loads, confirm the pagination footer shows the expected total:
    const text = document.body.innerText.match(/(d+)–(d+) of (d+)/);
    // text[3] = total number of playlists
  Note the total and calculate the number of pages: Math.ceil(total / 50).

STEP 2 — EXTRACTION FUNCTION:
  Run this function on each page to collect playlist data.
  Each page yields up to 50 rows via the <ytcp-playlist-row> custom element.

  function extractPlaylists() {
    const rows = Array.from(document.querySelectorAll('ytcp-playlist-row'));
    return rows.map(row => {
      // Title
      const titleEl = row.querySelector('a.remove-default-style');
      const title = titleEl ? titleEl.textContent.trim() : '';

      // Shareable link — built from playlist ID in the Studio href
      const studioHref = titleEl ? titleEl.href : '';
      const idMatch = studioHref.match(/\/playlist\/([^\/]+)\//);
      const playlistId = idMatch ? idMatch[1] : '';
      const shareableLink = playlistId
        ? 'https://www.youtube.com/playlist?list=' + playlistId
        : '';

      // Description
      const descEl = row.querySelector('div.description');
      let description = descEl ? descEl.textContent.trim() : '';
      description = description.replace(/\s+/g, ' ').trim();

      // Video count
      const countEl = row.querySelector('.videos-count-cell');
      const videoCount = countEl ? countEl.textContent.trim() : '';

      return { title, shareableLink, description, videoCount };
    });
  }

STEP 3 — PAGINATE AND ACCUMULATE:
  window._playlistData = [];

  // On each page:
  //   1. Run extractPlaylists() and concat to window._playlistData
  //   2. Click the Next Page button to advance
  //   3. Wait ~2 seconds for the new page to render
  //   4. Repeat until the page range footer shows you are on the last page

  // Clicking Next Page:
  document.getElementById('navigate-after').click();

  // The button element is: <ytcp-icon-button id="navigate-after">
  // It is NOT disabled on non-final pages (check aria-disabled="false").
  // Stop paginating when the range shows e.g. "201–225 of 225"
  // (i.e. when the end of the range equals the total).

STEP 4 — VERIFY COUNT:
  Confirm window._playlistData.length equals the total from Step 1.
  Also verify all rows have non-empty title, shareableLink, and videoCount.

STEP 5 — BUILD AND DOWNLOAD TSV:
  const escape = s => (s || '').replace(/\t/g, ' ').replace(/\n/g, ' ').replace(/\r/g, '');
  const header = ['Title', 'Description', 'Shareable Link', 'Video Count'];
  const rows = window._playlistData.map(d =>
    [escape(d.title), escape(d.description), escape(d.shareableLink), escape(d.videoCount)].join('\t')
  );
  const tsv = [header.join('\t'), ...rows].join('\n');

  const blob = new Blob([tsv], { type: 'text/tab-separated-values' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = 'youtube_playlists.tsv';
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);

KEY DOM SELECTORS (YouTube Studio):

  Description:         div.description                 (inside each row)
  Video count:         div.videos-count-cell           (inside each row)
  Rows-per-page:       ytcp-select#page-size
  Next page button:    ytcp-icon-button#navigate-after
  Page range text:     body.innerText matched against  /(d+)–(d+) of (d+)/

  - The channel ID (UCLk1BKTtXoyFD068_Ob2pcw) is embedded in the URL and

WORKFLOW 2: Setlist.fm Attended Concerts → TSV
================================================================================

URL:
  https://www.setlist.fm/attended/dan2bit

PURPOSE:
  Extract all attended concert setlist entries (date, band name, description,
  URL) into a TSV file. The total count will vary; do not hard-code 401.

PRE-REQUISITE — EXPAND ENTRIES PER PAGE:
  Setlist.fm paginates attended concerts. Before extracting, ensure ALL entries
  are loaded on a single page by selecting the maximum items-per-page option
  (look for a "per page" or pagination size control on the page).
  Verify that the page shows all entries in a single scrollable list before
  running the extraction script — if pagination is still present, step through
  each page using the same accumulation pattern as Workflow 1.

  IMPORTANT: The extraction script below reads from the already-rendered DOM.
  All entries must be present in the DOM before running it.

STEP 1 — COUNT UNIQUE SETLIST LINKS:
  Run this check first. If the count does not equal the known total, do not
  proceed — resolve the pagination/loading issue first.

  const allLinks = Array.from(document.querySelectorAll('a'));

STEP 2 — EXTRACTION FUNCTION:
  Each attended concert is rendered as <li class="setlist"> in a
  <ul class="list-unstyled"> container.

  function extractEntry(li) {
    // URL: the date block is wrapped in an anchor linking to the setlist page
    const dateA = li.querySelector('div.column.date a');
    const url = dateA ? dateA.href : '';

    // Date: month/day/year are inside span.smallDateBlock within the date anchor
    const dateBlock = dateA ? dateA.querySelector('span.smallDateBlock') : null;
    let date = '';
    if (dateBlock) {
      const month = dateBlock.querySelector('strong.text-uppercase')?.textContent.trim() || '';
      const day   = dateBlock.querySelector('strong.big')?.textContent.trim() || '';
      // Year: the bare <span> (no class) inside smallDateBlock
      const yearSpan = Array.from(dateBlock.querySelectorAll('span'))
        .find(s => s !== dateBlock && !s.className);
      const year = yearSpan?.textContent.trim() || '';
      date = month + ' ' + day + ' ' + year;
    }

    // Band name: first <strong> inside div.setlistLink that is NOT inside
    // div.column.date (i.e. skip the month/day strongs)
    const setlistLinkDiv = li.querySelector('div.setlistLink');
    let bandName = '';
    if (setlistLinkDiv) {
      const dateColDiv = setlistLinkDiv.querySelector('div.column.date');
      const allStrongs = Array.from(setlistLinkDiv.querySelectorAll('strong'));
      for (const s of allStrongs) {
        if (!dateColDiv?.contains(s) && s.textContent.trim()) {
          bandName = s.textContent.trim();
          break;
        }
      }
    }

    // Description: span.subline (usually "Venue, City, State, Country"
    // but sometimes an event/festival name)
    const sublineEl = li.querySelector('span.subline');
    const description = sublineEl ? sublineEl.textContent.trim() : '';

    return { date, url, bandName, description };
  }

  // Run across all entries:
  const liItems = Array.from(document.querySelectorAll('li.setlist'));
  const allData = liItems.map(extractEntry);

STEP 3 — VERIFY COUNT:
  Confirm allData.length equals uniqueHrefs.length from Step 1.
  Also confirm zero entries have empty url, date, or bandName.

  const blob = new Blob([tsv], { type: 'text/tab-separated-values' });

  const a = document.createElement('a');
  a.href = url2;
  a.download = 'setlist_attendances.tsv';
  document.body.appendChild(a);
  a.click();

KEY DOM SELECTORS (setlist.fm):
  Concert entry rows:  li.setlist
  Date anchor:         div.column.date a               (inside each li)
  Date block:          span.smallDateBlock             (inside date anchor)
  Month:               strong.text-uppercase           (inside smallDateBlock)
  Day:                 strong.big                      (inside smallDateBlock)
  Year:                span (no class)                 (inside smallDateBlock)
  Band name:           first strong NOT in div.column.date, inside div.setlistLink
  Description:         span.subline                    (inside each li)

NOTES:
  - Each setlist entry generates TWO anchor tags pointing to the same URL
    (one for the date block, one elsewhere in the row). Always deduplicate
    hrefs when doing the pre-flight count check.
  - The description field is whatever is in span.subline. For regular concerts
    this is "Venue, City, State, Country". For festival/tribute shows it may
    be an event name instead — this is correct behaviour, not an error.
  - The username segment of the URL (dan2bit) is specific to this account.
